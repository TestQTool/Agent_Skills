# Skill: Explore — Discover Locators and Map App Structure

## When to use this skill
Before generating any test scripts for a new feature/module.
Read this skill first when the user says:
"explore the login page", "find locators for X", "map the leave form", etc.

---

## Goal
Identify reliable, stable locators for a feature and produce a ready-to-use
`<Feature>PageObjects.java` stub with real selectors.

---

## Step-by-Step Process

### Step 1 — Read app context
Read `app-context.md` to understand the app, URL, and known quirks.

### Step 2 — Navigate and inspect
Open the target URL (BASE_URL from config.properties + feature path).
Inspect each interactive element. Prioritize locators in this order:

**Locator Priority (highest to lowest):**
1. `By.id("...")` — most stable, prefer always
2. `By.cssSelector("[data-testid='...']")` — second best
3. `By.cssSelector("[name='...']")` — for form inputs
4. `By.cssSelector("[aria-label='...']")` — for accessibility-labeled elements
5. `By.cssSelector(".class-name")` — only if class is stable (not generated)
6. `By.xpath("//tag[@attr='val']")` — last resort
7. NEVER: `By.xpath("/html/body/div[3]/div[2]/...")` — absolute XPath is forbidden

### Step 3 — Group locators by UI section
Map what you find into these sections:
- Page heading/title
- Form inputs (text, select, date, checkbox, radio)
- Buttons (submit, cancel, save, delete, edit, filter)
- Error/success messages
- Table (header, row, cell, pagination, no-data message)
- Modal/dialog (if present)
- Navigation/breadcrumb

### Step 4 — Generate PageObjects file

Output format: `pageObjects/<Feature>PageObjects.java`

```java
package com.qualitymatrix.pageObjects;

import org.openqa.selenium.By;

/**
 * <Feature>PageObjects — Locators only. Zero logic. Zero methods.
 * Generated by: ExploreAgent | Framework: QualityMatrix/selenium-java-bdd
 * Verified against: <APP_NAME> <URL> on <DATE>
 */
public class <Feature>PageObjects {

    // ── Page Heading ───────────────────────────────────────────────────────
    public static final By PAGE_HEADING       = By.cssSelector("h1");
    public static final By PAGE_SUB_HEADING   = By.cssSelector("h2");

    // ── Form Inputs ────────────────────────────────────────────────────────
    public static final By PRIMARY_INPUT      = By.id("primaryField");

    // ── Buttons ────────────────────────────────────────────────────────────
    public static final By SUBMIT_BTN         = By.cssSelector("button[type='submit']");
    public static final By CANCEL_BTN         = By.cssSelector("button[type='button']");

    // ── Messages ───────────────────────────────────────────────────────────
    public static final By SUCCESS_MESSAGE    = By.cssSelector(".success-message");
    public static final By ERROR_MESSAGE      = By.cssSelector("[role='alert']");

    // ── Table ──────────────────────────────────────────────────────────────
    public static final By TABLE_BODY         = By.cssSelector("table tbody");
    public static final By TABLE_ROW          = By.cssSelector("table tbody tr");

    // ── UNVERIFIED — update after exploration on live app ──────────────────
    // public static final By UNKNOWN_ELEMENT = By.id("TODO");
}
```

### Step 5 — Flag unstable locators
If any locator uses a generated class (e.g. `css-xk3d2f`) or dynamic ID,
add a comment: `// ⚠ UNSTABLE — verify after each deploy`

### Step 6 — Output summary
After generating the file, report:
- How many locators found per section
- Which locators are UNVERIFIED (need live app check)
- Any dynamic elements that need special handling (waitForElement, JS click, etc.)
- Recommended WebActions methods for each interaction type
