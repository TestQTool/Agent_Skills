# Onboarding Guide - Selenium Python BDD

## Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Python | 3.10+ | https://www.python.org/downloads/ |
| pip / venv | bundled with Python | python -m venv .venv |
| Git | Any | https://git-scm.com |
| Chrome / Edge / Firefox | Latest | Selenium Manager handles drivers |
| VS Code / PyCharm | Any | Optional; agent works from CLI |

No manual chromedriver download is normally needed. Selenium Manager handles browser drivers for Selenium 4.6+.

---

## Step 1 - Clone the Static Framework into Client Repo

```bash
git clone https://github.com/TestQTool/StaticFrameworks.git
cp -r "StaticFrameworks/Web Automation/selenium-python-bdd" <client-project>/
cd <client-project>/selenium-python-bdd
```

## Step 2 - Create Virtual Environment

```bash
python -m venv .venv
.venv\Scripts\activate
source .venv/bin/activate
```

## Step 3 - Install Dependencies

```bash
pip install -r requirements.txt
```

## Step 4 - Configure

```bash
cp .env.template .env
cp config/config.yaml.template config/config.yaml
```

Set at minimum:

```yaml
base_url: https://your-app.com
browser: chrome
headless: false
```

## Step 5 - Run

```bash
behave
behave --tags=@smoke
behave --tags=@regression
behave --tags=@login
behave --tags=@TC-LM-001
behave -D browser=firefox -D headless=true
behave -D environment=staging -D base_url=https://staging.app.com
```

If the framework exposes pytest wrappers, use the project-provided command, commonly:

```bash
pytest tests/ -m smoke
```

## Step 6 - View Reports

```bash
allure serve reports/allure-results
```

## Folder Map - What the Agent Generates vs What Exists

```text
selenium-python-bdd/
|-- core/ or framework/                <- DO NOT TOUCH (framework)
|-- utils/                             <- DO NOT TOUCH unless explicitly owned
|-- hooks/ or features/environment.py  <- DO NOT TOUCH (framework)
|-- page_objects/                      <- AGENT GENERATES HERE
|-- pages/                             <- AGENT GENERATES HERE
|-- features/
|   |-- *.feature                      <- AGENT GENERATES HERE
|   `-- steps/                         <- AGENT GENERATES HERE
|-- config/                            <- configure only when requested
`-- reports/                           <- generated by test runs
```
