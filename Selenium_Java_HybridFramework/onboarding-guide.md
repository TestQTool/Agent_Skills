## Step 3 — Run the Automation Workflow

Each feature follows this 5-command workflow:

```bash
# 1. Start a Claude Code session
cd HybridFramework/
claude

# 2. Explore the feature (browse live app, capture locators)
/explore login

# 3. Generate test cases from exploration notes
/generate-tests login
# ★ REVIEW test-cases/login/test-cases.md before proceeding

# 4. Build Java scripts from approved test cases
/build-scripts login

# 5. Review generated files for standards compliance
/review login
```

---

## Recommended Module Order

| Step | Feature | Why |
|------|---------|-----|
| 1 | Login + Logout | Foundation — everything else depends on this |
| 2 | Dashboard / Home | Verifies post-login state and navigation |
| 3 | Core module | Primary business functionality |
| 4 | Supporting modules | Secondary workflows |
| 5 | Admin / Settings | User management, config |

## Adding a New Feature

```
/explore <feature>
  → Generates: test-cases/<feature>/exploration-notes.md

/generate-tests <feature>
  → Generates: test-cases/<feature>/test-cases.md
  → ★ REVIEW before next step

/build-scripts <feature>
  → Generates: src/test/java/pages/<Feature>Page.java
               src/test/java/testcases/<Feature>TC.java

/review <feature>
  → Generates: test-cases/<feature>/review-report.md
  → Auto-fixes violations if found
```

If locators break after a UI update:
```
/heal <feature>
  → Fixes @FindBy annotations in src/test/java/pages/<Feature>Page.java
  → Generates: test-cases/<feature>/heal-log.md
```

## Running Tests

```bash
# All tests
mvn clean test

# Specific class
mvn clean test -Dtest=LoginTC

# Specific group
mvn clean test -Dgroups=sanity
mvn clean test -Dgroups=regression

# Specific suite file
mvn clean test -DsuiteFile=testng.xml

# View reports
allure serve allure-results/
open Reports/ExtentReport.html
```

## Common Issues & Fixes

| Problem | Fix |
|---------|-----|
| `NullPointerException` on WebElement | `@FindBy` locator changed — run `/heal <feature>` |
| Test fails on first run | Check `Config.properties` — URL and credentials |
| Browser doesn't open | Check `Browser` value: `chrome` / `edge` / `firefox` |
| `SessionNotCreatedException` | Update ChromeDriver to match installed browser version |
| Allure report empty | Run `allure serve allure-results/` after test execution |
| Extent report missing | Check `Reports/` folder exists — auto-created on first run |
| `Thread.sleep` in existing code | Replace with `webActions.waitVisibilityOfElementLocated(...)` |
| Tests run on wrong environment | Check `appEnvironment` in `Config.properties` |
